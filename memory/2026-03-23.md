## Workflow (End-to-End)

1. **User (Herb)** says in Telegram: "Open Notepad"
2. **Atlas (me)** on VPS:
   - Parses command via `parseChainedCommand()`
   - Creates JSON: `{ type: "operation", operation: "app.launch", params: { appPath: "notepad.exe" } }`
   - Writes to file: `cmd-TIMESTAMP.json`
   - Commits to GitHub: `atlas-commands` repo
   - Pushes to origin master
3. **Watcher on Windows machine:**
   - Polls GitHub every 5 seconds
   - Detects new `cmd-*.json` file
   - Executes via HTTP POST to `http://127.0.0.1:9999/execute-operation`
   - Deletes command file after execution
4. **Result:** Action completes on Windows machine, Atlas reports success

## Key Decisions Made

1. **GitHub bridge over ngrok** — ngrok auth token issues; GitHub is more reliable and free
2. **PowerShell watcher** — More reliable than batch script on Windows
3. **JSON command queue** — Human-readable, easy to debug
4. **5-second polling** — Fast enough for responsiveness, light on resources
5. **Auto-delete after execution** — Prevents duplicate executions

## Constraints & Safety

- Commands must be explicitly queued by Atlas (no direct user HTTP)
- Approval model: Safe primitives auto-approve; operations logged for review
- Command timeout: 10000ms
- No loops or conditionals (sequential workflows only)
- All commands logged to GitHub history (audit trail)
